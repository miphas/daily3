

### 小程序运行原理

webview        JS Runtime
  |                 |
       Wx Bridge
  |                 |
缓存服务、网络服务、设备信息等等


### 小程序为什么会快  
1. 渲染由 webview 负责，与 JS 运行分开，JS执行不会阻塞渲染
2. 小程序预渲染机制，会初始化一些页面，页面跳转会比较快
3. 通过桥能力将数据缓存，二次打开判断无更新可以直接读取缓存（更新做的好也可以 patch 进行）
4. 通过桥能力可以优化网络，由于预先在官网配置会访问的域名，小程序可以提前 prefetch-dns 等
